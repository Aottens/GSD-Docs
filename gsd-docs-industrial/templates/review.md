## Resolution Approach

**Default (no --route-gaps flag):**
Findings are logged for manual resolution. Engineer addresses flagged issues and updates sections as needed.

**With --route-gaps flag:**
Flagged issues automatically routed to gap closure pipeline:
1. Flagged issues converted to gap format
2. Fix plans generated via plan-phase --gaps --source review
3. Engineer runs write-phase to execute fixes
4. Engineer runs verify-phase to re-check

**Gap closure preview:**
```
Review complete. {N} issues flagged.

Flagged issues:
1. [High] 03-03: Missing I/O signal DO-300-04
2. [Medium] 03-05: Unclear alarm response for AL-500-02
3. [Low] 03-06: State diagram could be clearer

Generate fix plans for these issues? (Y/n)
```

Engineer can deselect issues before routing to gaps.

## Validation Rules

**For review-phase workflow:**
1. Status values: Approved, Comment, Flag (no other values)
2. Severity values: High, Medium, Low (for flagged issues only)
3. Every flagged issue must have detailed entry in "Flagged Issues (Detail)" section
4. Comments don't require detailed entry (captured in feedback table only)
5. Review Progress updated after each session

**For plan-phase --gaps --source review:**
1. Read flagged issues from "Flagged Issues (Detail)" section
2. Extract severity, finding, suggested action
3. Generate fix plan per flagged issue (or group related issues)
4. Mark "Routed to gaps: Yes" after fix plans created

## Usage Notes

**Interactive section presentation:**

For each section:
1. Show SUMMARY.md (context)
2. Show cross-references (if any)
3. Show CONTENT.md (paginated if >50 lines)
4. Collect feedback via AskUserQuestion:
   - Approved (no issues)
   - Comment (minor note)
   - Flag (needs revision)
   - View full content
   - Skip to next

**Multi-session support:**

Large phases (>15 sections) may require review in multiple sessions to avoid fatigue. Review Progress section tracks position:
- Sections reviewed: 1-10 of 23
- Next: §3.11

Resume command: `/doc:review-phase {N} --resume` reads progress and skips already-reviewed sections.

**Pause points:**

After every 10 sections reviewed:
```
You've reviewed 10 of 23 sections. Continue or save progress?
  - Continue (review next batch)
  - Save and exit (resume later with --resume)
```
