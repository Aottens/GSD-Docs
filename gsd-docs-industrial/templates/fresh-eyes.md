## Severity Calibration

**MUST-FIX:** Genuinely confusing, blocks understanding or implementation
- Engineer: undefined value blocking implementation, inconsistent terminology causing confusion
- Customer: jargon blocking approval, scope commitment unclear
- Operator: missing alarm response, unclear recovery procedure

**SHOULD-FIX:** Improvement that would help clarity
- Engineer: reference to planning docs instead of summarizing in FDS, missing diagram that would clarify
- Customer: unexplained abbreviation, overly technical language
- Operator: missing state duration estimate, no manual override documented

**CONSIDER:** Nice-to-have, minor enhancement
- Engineer: could add Mermaid diagram, could reference related section
- Customer: could clarify response time, could add example
- Operator: could add typical sequence duration, could include troubleshooting tips

**Perspective-specific calibration:**
- Engineer: Stricter on safety-critical sections, lighter on overview/appendices
- Customer: Stricter on scope boundaries and commitments, lighter on technical details
- Operator: Stricter on alarm responses and manual procedures, lighter on architecture
