## Phase 1: Change

**Goal:** Document the change request, its reason, and scope boundaries.

**Description:**

## Phase 2: Implementation

**Goal:** Define technical changes, impact, and verification criteria.

**Description:**
- Technische wijzigingen: specific changes to equipment, software, or parameters
- Impact analyse: effects on existing system (referencing BASELINE.md)
- Test criteria: acceptance tests to verify the change

**Success Criteria:**
- Every technical change is specified at implementation level
- Impact on existing system is assessed and documented
- At least one acceptance test per functional change
