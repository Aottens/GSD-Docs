    Create the LLM provider abstraction that enables future local model support (SYST-04):

    1. **Abstract provider** (`app/llm/provider.py`):
       - LLMProvider(ABC) with:
         - `async def complete(self, messages: list[dict[str, str]], max_tokens: int = 4096, temperature: float = 0.7, **kwargs) -> str` — returns completion text
         - `async def stream_complete(self, messages: list[dict[str, str]], max_tokens: int = 4096, temperature: float = 0.7, **kwargs) -> AsyncGenerator[str, None]` — yields text chunks
         - `def get_model_name(self) -> str` — returns the model identifier
         - `async def health_check(self) -> bool` — returns True if provider is accessible (for /api/health integration)

    2. **LiteLLM implementation** (`app/llm/litellm_provider.py`):
       - LiteLLMProvider(LLMProvider) with:
         - `__init__(self, model: str, api_key: str | None = None)` — stores model name, optionally sets api_key
         - `complete()`: Use `litellm.acompletion()` with explicit max_tokens (Pitfall 7 from research). Return `response.choices[0].message.content`.
         - `stream_complete()`: Use `litellm.acompletion()` with stream=True. Yield `chunk.choices[0].delta.content` for non-None deltas.
         - `get_model_name()`: Return self.model
         - `health_check()`: Try a minimal completion (1 token max) and return True/False. Catch all exceptions, return False on failure.
       - Wrap LiteLLM calls in try/except, raise a custom `LLMProviderError` exception with clear error messages for: auth failures, rate limits, model not found, and generic errors.

    3. **LLM service** (`app/services/llm_service.py`):
       - `get_llm_provider() -> LLMProvider`: Factory function that reads settings (LLM_PROVIDER, LLM_MODEL, ANTHROPIC_API_KEY) and returns the appropriate provider instance. Currently only returns LiteLLMProvider but structured for easy addition of other providers.
       - `LLMProviderError(Exception)`: Custom exception class with provider name, error type, and user-friendly message.

    4. **Wire into app**: Update `app/llm/__init__.py` to export LLMProvider and get_llm_provider. Optionally extend health endpoint to report LLM provider status (but don't fail health check if LLM is unavailable — app should start without API key for development).
