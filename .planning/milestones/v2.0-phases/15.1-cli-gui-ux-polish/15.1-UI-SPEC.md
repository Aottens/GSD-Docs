## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. `gap-1` in CliCommandBlock) |
| sm | 8px | Compact element spacing (e.g. button gaps, badge rows) |
| md | 16px | Default element spacing (e.g. card padding, `gap-4`) |
| lg | 24px | Section padding (e.g. Card `p-6` = 24px) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch target minimum 44px for icon-only ghost buttons (CliCommandBlock copy button uses `h-6 w-6` — acceptable since it is always adjacent to a text target).

Source: Tailwind v4 default scale + existing component inspection (`CliCommandBlock.tsx`, `ProjectOverview.tsx`).

## Interaction Contracts

### Contract 1: Change Notification Toasts

**Trigger:** Polled query data changes between poll intervals (5s setup-state, 10s phase timeline, 15s document outline).

**Behavior:**
- On initial data load: silent — no toast. Set `prevRef.current` to initial data and return.
- On subsequent data change: fire `toast.info(title, { description })` once per change event.
- On re-render without data change: no toast (useRef prevents duplicate fires).

**Visual:**
- Toast position: bottom-right (sonner default — already configured in `App.tsx`).
- Toast variant: `toast.info` (blue info icon in light mode; matches sonner's neutral info style).
- Toast duration: sonner default (4 seconds).
- No dismiss button required — auto-dismisses.

**Copy (Dutch):**

| Data source | Title | Description |
|-------------|-------|-------------|
| setup-state | `Setup status bijgewerkt` | `CLI heeft wijzigingen aangebracht in de projectconfiguratie.` |
| phase timeline | `Fase status bijgewerkt` | `Een of meer fases hebben een nieuwe status na CLI werk.` |
| document outline | `Documentstructuur bijgewerkt` | `Nieuwe secties beschikbaar na CLI schrijffase.` |

**Fingerprint approach:**
- setup-state: `doc_types.map(d => "${d.id}:${d.status}:${d.file_count}").join(',')`
- phase timeline: `phases.map(p => "${p.number}:${p.status}").join(',')`
- document outline: `"${sections.length}:${countWithContent(sections)}"`

### Contract 2: "Bekijk documenten" Link in PhasePopover

**Placement:** Inside `PopoverContent`, after the CLI Command section. Rendered conditionally.

**Condition:** `phase.has_content === true` AND `onNavigateToDocs` prop is defined.

**Visual:**
- Element: `<button>` (not `<a>`)
- Styling: `text-xs text-primary hover:underline`
- Label: `Bekijk documenten voor fase {phase.number} →`
- No icon — text link only, consistent with popover's compact layout.

**Behavior:** On click — call `onNavigateToDocs()` and close popover (`setIsOpen(false)`). No loading state. No confirmation.

**Prop threading path:** `ProjectWorkspace` → `PhaseTimeline` → `PhaseNode` → `PhasePopover`. Prop name: `onNavigateToDocs?: () => void`.

### Contract 3: "Referenties uploaden" Quick Action

**Before:** Disabled button with subtitle "Fase 9".
**After:** Enabled button, no subtitle, navigates to Referenties tab.

**Visual:**
- Remove `disabled` attribute.
- Remove `title="Beschikbaar in fase 9"` tooltip.
- Remove `<span className="text-xs text-muted-foreground">Fase 9</span>` sub-label.
- Keep: `variant="outline"`, `className="h-auto py-4 flex-col gap-2"`, `<Upload className="h-5 w-5" />`, `<span className="text-sm">Referenties uploaden</span>`.

**Behavior:** `onClick={() => onNavigate?.('references')}`. No loading state. No confirmation.

### Contract 4: Quick Actions Grid — 2-Column Layout

**Before:** `grid grid-cols-1 sm:grid-cols-3 gap-3` (3 buttons).
**After:** `grid grid-cols-1 sm:grid-cols-2 gap-3` (2 buttons).

**Buttons remaining (in order):**
1. "Referenties uploaden" (Upload icon, now wired)
2. "Documenten bekijken" (FileText icon, existing `onNavigate?.('documents')`)

### Contract 5: CliCommandBlock Canonical Visual

Shared `CliCommandBlock` from `@/features/timeline/components/CliCommandBlock.tsx` is the only allowed implementation.

**Visual:**
- Container: `flex items-center gap-2 bg-muted rounded px-3 py-2`
- Code text: `text-xs font-mono flex-1 text-foreground`
- Copy button: `Button` ghost, `size="icon"`, `h-6 w-6 shrink-0`
- Copy icon: `Copy` (lucide), `h-3 w-3`

**EmptySectionCard note:** Previously used `text-sm font-mono`. After deduplication, `text-xs font-mono` is canonical. This is an intentional visual alignment, not a bug.
