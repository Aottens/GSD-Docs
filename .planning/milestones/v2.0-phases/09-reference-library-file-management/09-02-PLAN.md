**2. Create upload hook** in `frontend/src/features/files/hooks/useFileUpload.ts`:

Follow the research pattern from 09-RESEARCH.md "Pattern 3: Chunked Upload with Progress Tracking".

`useFileUpload(projectId?: number)` hook:
- State: `progressMap: Map<string, UploadProgress>` (keyed by file name)
- Uses XMLHttpRequest (not fetch) for `upload.onprogress` events — this gives real per-file progress bars
- Upload endpoint: `projectId ? /api/projects/${projectId}/files : /api/files/shared`
- Optional `folderId` parameter passed as query string
- `uploadFiles(files: File[], folderId?: number)`: Initializes all files as 'pending', uploads sequentially (avoid overwhelming server)
- Per-file progress flow: pending -> uploading (progress 0-100%) -> done OR error
- On XHR error: parse response JSON for `detail` field to show specific validation error message
- On network error: set status 'error' with "Netwerkfout" message
- Returns: `{ uploadFiles, progressMap, isUploading, clearProgress }`
- `clearProgress()`: Clears the progress map (call after dismissing upload results)

IMPORTANT: The upload uses `FormData` with `file` field (matching backend's `UploadFile` parameter name). Do NOT set Content-Type header on XHR — browser sets multipart boundary automatically.

**3. Create file listing hook** in `frontend/src/features/files/hooks/useFiles.ts`:

Use React Query (TanStack Query) following the pattern from `frontend/src/features/projects/queries.ts`.

- `useProjectFiles(projectId: number, folderId?: number, fileType?: string, search?: string)`: GET `/api/projects/${projectId}/files` with query params
- `useSharedFiles(folderId?: number, fileType?: string, search?: string)`: GET `/api/files/shared` with query params
- `useProjectFolders(projectId: number)`: GET `/api/projects/${projectId}/folders`
- `useSharedFolders()`: GET `/api/files/shared/folders`
- `useCreateFolder(projectId: number)`: POST mutation to `/api/projects/${projectId}/folders`
- `useDeleteFile()`: DELETE mutation to `/api/files/${fileId}`, invalidates file queries
- `useUpdateFile()`: PATCH mutation to `/api/files/${fileId}`, invalidates file queries
- `useReplaceFile()`: PUT mutation with FormData (similar to upload)
- `useOverrideFile()`: POST mutation to `/api/files/${fileId}/override`

For mutations that send FormData (replace, override), update the `api` client: Add a `postForm<T>(url: string, formData: FormData)` method to `frontend/src/lib/api.ts` that sends FormData without `Content-Type: application/json` header (let browser set multipart boundary). Also add `putForm<T>` for replace.
