**2. Create `backend/app/services/file_validator.py`** — Defense-in-depth validation pipeline.

Follow the research pattern from 09-RESEARCH.md "Pattern 1: Defense-in-Depth File Validation".

`async def validate_file_upload(file: UploadFile, settings: Settings) -> dict`:
- **Layer 1 — Extension whitelist**: Extract extension via `Path(filename).suffix.lower()`, check against `settings.ALLOWED_EXTENSIONS`. Raise HTTPException 400 with specific message: `"Bestandstype '{ext}' is niet toegestaan. Toegestaan: {', '.join(settings.ALLOWED_EXTENSIONS)}"` (Dutch per UI decision).
- **Layer 2 — Filename sanitization**: Use `os.path.basename(filename)`. If result differs from original, raise HTTPException 400: `"Ongeldige bestandsnaam (padtraversal gedetecteerd)"`.
- **Layer 3 — File size limit**: Read content via `await file.read()`. Check `len(content) > settings.MAX_UPLOAD_SIZE`. Raise HTTPException 413: `f"Bestand overschrijdt de maximale grootte van {settings.MAX_UPLOAD_SIZE // (1024*1024)}MB"`.
- **Layer 4 — Magic number validation**: Use `magic.from_buffer(content, mime=True)`. Check against `settings.ALLOWED_MIME_TYPES`. Raise HTTPException 400: `f"Bestandsinhoud '{mime}' komt niet overeen met extensie '{ext}'"`.
- **Layer 5 — Image verification** (only if mime starts with 'image/'): Use `PIL.Image.open(io.BytesIO(content))` then `img.verify()`. Catch exceptions and raise HTTPException 400: `"Ongeldig of beschadigd afbeeldingsbestand"`.
- Reset file position: `await file.seek(0)`.
- Return dict: `{'safe_filename': safe_filename, 'mime_type': mime, 'size': len(content), 'extension': ext}`.

Handle the case where python-magic is not installed (libmagic missing): catch ImportError, fall back to trusting the extension-based MIME type with a warning log. This prevents blocking development on machines without libmagic.

**3. Create `backend/app/services/file_storage.py`** — Filesystem operations.

`async def ensure_upload_dir(settings: Settings)`: Create `settings.UPLOAD_DIR` directory if it doesn't exist (called during app startup).

`async def save_file(file: UploadFile, scope: FileScope, project_id: int | None, folder_path: str, file_uuid: str, ext: str, settings: Settings) -> str`:
- Generate storage path: `{scope.value}/{project_id or 'shared'}/{folder_path}/{file_uuid}{ext}`
- Create parent directories with `os.makedirs(full_path.parent, exist_ok=True)`
- Write file content using aiofiles: `async with aiofiles.open(full_path, 'wb') as f: await f.write(await file.read())`
- Reset file position after read
- Return relative storage_path (relative to UPLOAD_DIR)

`async def delete_file(storage_path: str, settings: Settings) -> bool`:
- Construct full path from UPLOAD_DIR + storage_path
- Remove file if exists, return True/False
- Note: Called only for hard delete; soft delete just sets is_deleted flag

`async def replace_file(old_storage_path: str, file: UploadFile, scope: FileScope, project_id: int | None, folder_path: str, file_uuid: str, ext: str, settings: Settings) -> str`:
- Save new file (reuse save_file)
- Delete old file
- Return new storage_path

`def get_absolute_path(storage_path: str, settings: Settings) -> Path`:
- Return `Path(settings.UPLOAD_DIR) / storage_path`

**5. Create `backend/app/api/files.py`** — File endpoints.

Follow pattern from `backend/app/api/projects.py` (router, service injection, error handling).

Router prefix: `/api` (endpoints below include full paths)

**Project file endpoints:**
- `POST /api/projects/{project_id}/files`: Upload file to project. Accept `UploadFile`, optional `folder_id` query param. Call `validate_file_upload()`, then `save_file()`, then `FileService.create_file()`. Return `FileUploadResponse` with 201 status.
- `GET /api/projects/{project_id}/files`: List project files. Query params: `folder_id` (optional), `file_type` (optional: "pdf", "docx", "image"), `search` (optional). Return `FileListResponse`.
- `GET /api/files/{file_id}/download`: Download file. Look up file record, verify not deleted, return `FileResponse` from filesystem using FastAPI's `FileResponse` (from `starlette.responses`). Set `Content-Disposition: attachment; filename="{original_filename}"`.
- `GET /api/files/{file_id}/preview`: Serve file for inline preview. Same as download but with `Content-Disposition: inline`.
- `PATCH /api/files/{file_id}`: Update file metadata (rename via `original_filename`, move via `folder_id`). Return updated `FileResponse`.
- `PUT /api/files/{file_id}/replace`: Replace file content. Accept new `UploadFile`. Validate, save new, delete old from disk, update DB record (new size, mime_type, storage_path). Return `FileResponse`.
- `DELETE /api/files/{file_id}`: Soft-delete file. Set `is_deleted=True`, `deleted_at=now()`. Return 204.

**Shared library endpoints:**
- `POST /api/files/shared`: Upload file to shared library. Require admin auth (check `X-Admin-Key` header against `settings.ADMIN_API_KEY`; if ADMIN_API_KEY is empty, allow all — development mode). Accept `UploadFile`, optional `folder_id`. Return `FileUploadResponse`.
- `GET /api/files/shared`: List shared library files. Query params same as project files. No auth required (read-only for all).
- `DELETE /api/files/shared/{file_id}`: Soft-delete shared file. Require admin auth. Return 204.

**Override endpoint:**
- `POST /api/files/{file_id}/override`: Create project-specific override of a shared file. Body: `project_id`, `UploadFile`. Validates shared file exists, creates new project file with `overrides_file_id` pointing to original. Return `FileUploadResponse`.

**Admin auth dependency** (`get_admin_user`): Create a simple FastAPI `Depends` function that checks `X-Admin-Key` header against `settings.ADMIN_API_KEY`. If key is empty (not configured), allow through (dev mode). If key is set and doesn't match, raise 403. This is sufficient for 5-20 user scale per Claude's Discretion.
