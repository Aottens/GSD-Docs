**Issue 1A: Question loop / repetition (BLOCKER)**

Three changes to prevent question loops:

**1. Hard cap in `conversation_state.py` — `increment_question()` method:**

Change `increment_question()` to enforce a hard cap of 12 questions per topic as a safety valve. Current logic returns True at >= 4 (triggers check-in). Add a second threshold:
- At >= 4: return "check_in" (normal rhythm)
- At >= 12: return "force_check_in" (hard cap safety valve)

Update the method signature to return a string instead of bool:
```python
def increment_question(self) -> str:
    """Increment question counter. Returns 'continue', 'check_in', or 'force_check_in'."""
    self.questions_in_current_topic += 1
    if self.questions_in_current_topic >= 12:
        return "force_check_in"
    if self.questions_in_current_topic >= 4:
        return "check_in"
    return "continue"
```

**2. Foundation area detection fallback in `conversation_state.py`:**

Current `detect_covered_area()` uses keyword matching which may not trigger on all real Q&A content. Add a question-count-based fallback: if `questions_in_current_topic >= 2` and keyword match finds nothing, cycle through uncovered areas in order and mark the next one as covered. This ensures progress even when keywords don't match.

Add method:
```python
def detect_covered_area_with_fallback(self, question: str, answer: str) -> Optional[str]:
    """Keyword-match first, then fall back to question-count cycling."""
    # Try keyword match first
    area = self.detect_covered_area(question, answer)
    if area:
        return area
    # Fallback: after 2+ questions, cycle through uncovered areas
    if self.questions_in_current_topic >= 2:
        all_areas = list(self._FOUNDATION_AREA_KEYWORDS.keys())
        for a in all_areas:
            if a not in self.foundation_areas_covered:
                return a
    return None
```

**Issue 1B: Decision summary timing (MAJOR) — backend side**

Decisions should accumulate silently during Q&A. No `decision_captured` SSE events during discussion. All decisions shown at check-in.

```python
# REMOVE this block:
if decision:
    yield {"event": "decision_captured", "data": {"decision": decision}}
    state.decisions.append(decision)

# REPLACE with silent accumulation:
if decision:
    state.decisions.append(decision)
```

The `_build_checkin_message()` already shows decisions at check-in time (it filters `state.decisions` by topic). No change needed there.

```python
# In _handle_discussion(), where check-in is triggered:

    state.phase = ConversationPhase.check_in
    checkin_msg = self._build_checkin_message(state, language)

    # Get topic decisions for the Beslissingen tab
    topic_decisions = [
```
