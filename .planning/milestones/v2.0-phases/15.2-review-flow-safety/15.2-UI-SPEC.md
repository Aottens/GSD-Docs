## Interaction Contracts

### Contract 1: Review-Safety Warning Banner (AssemblyPipeline)

**Trigger:** `AssemblyPipeline` mounts or re-mounts. Reads localStorage using `Object.keys(localStorage).filter(k => k.startsWith('review-session-{projectId}-'))` to aggregate rejected counts across all phases.

**When banner shows:**
- `rejectedCount > 0` — at least one section has `afgewezen` status

**Banner variant:** `Alert` with `variant="default"` (not destructive — warning, not error)

**Banner content:**
- `AlertTitle`: "Secties afgewezen in review"
- `AlertDescription`: "{N} {sectie is / secties zijn} afgewezen. Overweeg de review af te ronden voordat u exporteert."
- Two action buttons (in `mt-3 flex gap-2`):
  1. `Button size="sm" variant="outline"` — "Terug naar documenten" — calls `onNavigateToDocs()` prop
  2. `Button size="sm" variant="ghost"` — "Toch doorgaan" — dismisses banner by setting local `dismissed` state

**Banner dismissal:** Local `useState(false)` for dismissed state. Banner hides when dismissed. Re-appears on next mount (tab switch + return).

**Banner position:** Below existing `isFinalBlocked` alert, above pipeline stages.

**Prop addition:** `AssemblyPipeline` gains `onNavigateToDocs?: () => void`. `ExportTab` receives this prop from `ProjectWorkspace` which calls `setActiveSection('documents')`.

**Does NOT block export:** The "Samenstellen starten" button remains enabled. This is a warning, not a hard block.

### Contract 2: Multi-Phase Selector in DocumentsTab

**When selector renders:** Only when `verifiedPhases.length > 1`. If only one verified phase exists, no selector is shown — no visual noise.

**Selector anatomy:** Thin toolbar (`shrink-0 border-b px-4 py-2`, height ~36px) above the split panel:
- Text label: `"Review fase:"` — `text-sm text-muted-foreground`
- `Select` component: `w-64 h-8 text-sm`
- `SelectItem` label format: `"Fase {N}: {phase.name}"`

**Layout fix:** Outer `div` in DocumentsTab wraps selector + panel in `flex flex-col h-full overflow-hidden`. The existing `ref={containerRef}` div changes from `h-full` to `flex-1 overflow-hidden`. This preserves drag-resize behavior.

**State ownership:** `DocumentsTab` owns `const [selectedPhaseNumber, setSelectedPhaseNumber] = useState<number | undefined>(activePhaseNumber)`.

**Sync behavior:** `useEffect` on `activePhaseNumber` prop — only sets `selectedPhaseNumber` when it is currently `undefined` (i.e. not yet user-selected). Once the engineer selects a phase manually, prop updates do not override the selection.

**ReviewProvider key:** Driven by `selectedPhaseNumber` (user-controlled), not polling. Eliminates re-mount instability.

### Contract 3: Persistence Indicator in ReviewSummary

**When shown:** When `allReviews.length > 0` (same guard already on ReviewSummary).

**What to add:** Below the export JSON button, add a static paragraph:
- `<p className="text-xs text-muted-foreground mt-2">Sessie opgeslagen in browser</p>`

**Visual treatment:** `text-xs text-muted-foreground` — understated, not a prominent UI element. Engineer reads it once and trusts persistence.
