## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (e.g. badge `px-1.5`) |
| sm | 8px | Compact element spacing (e.g. outline node `py-1.5`, section header gap) |
| md | 16px | Default element spacing (e.g. section content padding, button gap) |
| lg | 24px | Section padding, separation between content blocks |
| xl | 32px | Layout gaps between major document sections |
| 2xl | 48px | Major section breaks in document preview |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Outline node minimum touch target: 36px height (`min-h-[36px]`) ŌĆö carries over from Phase 11 OutlineNode pattern
- Outline node indent per depth level: 12px multiplier (`8 + depth * 12`) ŌĆö carries over from Phase 11

## Interaction Contracts

### 1. Review Controls Visibility

- Review buttons (Goedkeuren / Opmerking / Afwijzen) appear on leaf-node `SectionBlock` components when `phaseHasVerification === true`
- No "start review" button. No mode switch. Controls appear automatically.
- Source: CONTEXT.md decision ŌĆö "No explicit 'start review' action"

### 2. Feedback Textarea

- Textarea renders **below** review buttons only when engineer clicks "Opmerking" or "Afwijzen"
- Textarea is **hidden** (not mounted) when "Goedkeuren" is selected
- "Opslaan" button appears inline with textarea
- Feedback textarea and VerificationDetailPanel (Collapsible) are **sibling elements** ŌĆö never nested ŌĆö to prevent focus conflicts
- Source: CONTEXT.md decision on feedback capture; RESEARCH.md Pitfall 6

### 3. Verification Detail Panel

- Displayed as a `Collapsible` below section content
- Collapsed by default; engineer clicks to expand
- Shows: cycle badge (Cyclus N/2), truth-by-truth pass/gap list, gap descriptions with StandardsBadge pills
- Under a "Normen" sub-heading within the detail panel: standards violations only (Level 5)
- If phase `Status: GAPS_FOUND` and `current_cycle === max_cycles`: show `Alert` variant="destructive" with text "GEBLOKKEERD ŌĆö Maximaal aantal cycli bereikt. Neem contact op met de lead engineer." and CLI hint `/doc:plan-phase N --gaps`
- Source: CONTEXT.md decisions on verification display and standards compliance

### 4. Standards Reference Badges

- Render as subtle `Badge` variant="outline" pills inside VerificationDetailPanel
- Format: "ISA-88 ┬¦4.3" or "PackML ┬¦2.1"
- On hover: `Tooltip` shows full standard requirement text (expected vs found)
- Source: CONTEXT.md decision on standards compliance display

### 5. Outline Node Badge Evolution

| Backend status | Review context status | Display badge | Icon | Color |
|---|---|---|---|---|
| any | goedgekeurd | Ō£ō | CheckCircle2 | green-500 |
| any | opmerking | ŌÜĀ | MessageSquare | amber-500 |
| any | afgewezen | Ō£Ś | XCircle | red-500 |
| verified | none | V1 | CheckCircle2 | green-500 |
| written | none | G1 | FileText | amber-500 |
| planned | none | (wave badge) | Clipboard | blue-500 |
| empty | none | ŌĆö | Circle | muted-foreground |

Review status from `ReviewContext` takes precedence over backend status. `OutlineNode` TypeScript type does NOT change ŌĆö review status stays frontend-only in context.
Source: CONTEXT.md decision + RESEARCH.md Pattern 5

### 6. Post-Review Summary

- Renders in `ContentPanel` footer after all leaf sections are reviewed
- Shows: "X goedgekeurd ┬Ę Y opmerking ┬Ę Z afgewezen"
- Shows CLI command: `/doc:review-phase N`
- Shows "Exporteer als JSON" button ŌĆö copies review session JSON to clipboard via `navigator.clipboard.writeText()`
- Source: CONTEXT.md decision; RESEARCH.md Pattern code example

### 7. Loading and Error States

- Verification data loading: `Skeleton` component, height 48px (`h-12`), full width
- Verification data error: `text-sm text-muted-foreground` paragraph ŌĆö "Verificatiedata kon niet worden geladen. Ververs de pagina of controleer de verbinding."
- No verification file present: verification bar and review controls do not render (controlled by `phaseHasVerification` prop)
- Source: RESEARCH.md Claude's Discretion ŌĆö loading/error states

### 8. Animation / Transition

- Review controls appear with `transition-all duration-150` fade-in when `phaseHasVerification` becomes true
- Feedback textarea expand: CSS transition via `Collapsible` (already handles animation)
- No custom spring animations for review controls ŌĆö Motion library not used in this phase
- Source: RESEARCH.md Claude's Discretion ŌĆö animation/transition
