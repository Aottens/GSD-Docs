  <action>
    Read the SPECIFICATION.md (C:\Users\Aotte\Documents\Projects\GSD-Docs\SPECIFICATION.md) section 3.3 for the exact ROADMAP structure per type. Create 4 template files.

    **Template composition pattern (Pitfall 5 mitigation):**
    - Each template is a standalone ROADMAP skeleton — NOT a copy of a base with modifications
    - Templates use placeholder variables wrapped in curly braces: `{PROJECT_NAME}`, `{CLIENT}`, `{DATE}`, `{LANGUAGE}`, `{PHASE_COUNT}`
    - Each phase within a template has: Goal, Description (2-3 bullet points of what this phase covers), Success Criteria (2-3 observable outcomes), Dependencies (which prior phase must complete)
    - Templates are SMALL (50-100 lines each). They define STRUCTURE not content. The `/doc:new-fds` command fills in project-specific details.
    - Duplication between templates is acceptable at the ROADMAP level because each type has genuinely different phase structures. The anti-duplication effort goes into the command logic, not the templates.

    **Type A — Nieuwbouw + Standaarden (6 phases):**
    From SPECIFICATION section 3.3:
    - Phase 1: Foundation (introductie, definities, referenties, standaarden scope PackML/ISA-88)
    - Phase 2: System Architecture (systeem overzicht, equipment hierarchy ISA-88, operating modes PackML)
    - Phase 3: Equipment Modules (per EM: beschrijving, states, parameters, interlocks — largest phase)
    - Phase 4: Control & HMI (control philosophy, HMI requirements, screen descriptions)
    - Phase 5: Interfaces & Safety (external interfaces, safety requirements, interlocks overview)
    - Phase 6: Appendices (signal list, parameter list, state transition tables)

    **Type B — Nieuwbouw Flex (4-5 phases):**
    - Phase 1: Foundation (introductie, definities, scope — no standards section)
    - Phase 2: System Overview (system description, functional blocks, process flow)
    - Phase 3: Functional Units (per unit: description, operation, parameters, interlocks — flexible structure)
    - Phase 4: HMI & Interfaces (operation, external connections, communication)
    - Phase 5: Appendices (optional — signal list, parameter list)

    **Type C — Modificatie Groot (3-4 phases):**
    - Phase 1: Scope & Baseline (change description, BASELINE.md reference, what changes YES/NO)
    - Phase 2: Delta Functional (changed functionality, new equipment/functions, impact on existing system)
    - Phase 3: Delta HMI & Interfaces (changed screens, new screens, interface changes)
    - Phase 4: Verification & Appendices (test criteria, regression check, updated signal list delta)

    **Type D — Modificatie Klein / TWN (2 phases):**
    - Phase 1: Change (change description, reason, scope in/out)
    - Phase 2: Implementation (technical changes, impact analysis, test criteria)

    Each template should have:
    - A header block with `{PROJECT_NAME}`, `{CLIENT}`, `{DATE}`, `{PROJECT_TYPE}`, `{LANGUAGE}`
    - A `## Overview` section with a one-paragraph description of this project type
    - Phase entries in the format used by GSD ROADMAP (Goal, Requirements mapping placeholder, Success Criteria, Dependencies)
    - A `## Progress` table at the bottom
    - Footer: `*Roadmap created: {DATE}*`

    Type C and D templates MUST include a reference to BASELINE.md in Phase 1 scope.
  </action>
