**Step 4: Create backend/app/services/assembly_service.py**
Implement FDS assembly service following complete-fds.md workflow:

- `async def check_assembly_readiness(project_dir: Path, mode: str) -> AssemblyReadinessSchema`:
  - For "final" mode: scan `.planning/phases/` — every phase dir with a `*-SUMMARY.md` must also have a `*-REVIEW.md`. Return unreviewed phase names.
  - For "draft" mode: at least one `*-SUMMARY.md` must exist anywhere. Return `ready=True` if found.
  - Return `has_content=False` if no SUMMARY.md files found.

- `async def assemble_fds(project_dir: Path, language: str, mode: str, v1_docs_path: Path, on_step: Callable) -> Path`:
  - Step 1 (on_step("Cross-referenties oplossen", 0)): Load fds-structure.json from `v1_docs_path/templates/fds-structure.json`. Parse project type from `project_dir/.planning/PROJECT.md` YAML frontmatter (type field). Build section order from fds-structure.json, selecting title by language ("en" or "nl" key).
  - Step 2 (on_step("Secties samenvoegen", 1)): Walk `.planning/phases/` directories in sorted order. For each `*-SUMMARY.md`, extract section content. Order by section IDs from fds-structure.json tree (depth-first). Build a single assembled markdown with IEC-style hierarchical numbering (`# 1 Title`, `## 1.1 Subtitle`, etc.).
  - Cross-reference resolution: scan assembled markdown for `{ref:X.Y}` patterns, replace with `Section X.Y`. Scan for `{fig:N}` patterns, replace with `Figure N`.
  - Step 3 (on_step("DOCX genereren", 2)): handled by export_service, not this function.
  - If mode == "draft": prepend `> **CONCEPT — Niet definitief**\n\n` at the top of the assembled markdown (visible header, not background watermark per RESEARCH.md Pitfall 6).
  - Write assembled markdown to `project_dir/output/FDS-assembled-{language}.md`. Create `output/` dir if it doesn't exist.
  - Return the Path to the assembled markdown file.

- `_parse_project_type(project_dir: Path) -> str`: Read PROJECT.md, parse YAML frontmatter, extract `type` field. Default to "A" if missing.

- `_load_fds_structure(v1_docs_path: Path) -> dict`: Load and return fds-structure.json.

- `_build_section_tree(fds_structure: dict, project_type: str, language: str) -> list[tuple[str, str, int]]`: Return ordered list of (section_id, title, depth) tuples using depth-first traversal. Handle Type C/D baseline section insertion (same logic as documents.py `_build_outline_sections`). Select title by language key.

**Step 5: Create backend/app/services/export_service.py**
Implement DOCX export service following export.md workflow:

- `def detect_pandoc() -> tuple[bool, str | None]`: Use `shutil.which("pandoc")` to check availability. Run `subprocess.run(["pandoc", "--version"], capture_output=True, text=True, timeout=5)` — parse version from first line. Return `(found, version)`.

- `async def export_to_docx(assembled_md: Path, project_dir: Path, mode: str, language: str, v1_docs_path: Path, on_step: Callable) -> str`:
  - on_step("DOCX genereren", 2): Invoke Pandoc via `asyncio.create_subprocess_exec`.
  - Pandoc flags: `--from markdown+yaml_metadata_block+pipe_tables+grid_tables+implicit_figures --to docx --standalone --toc --toc-depth=3 --number-sections --resource-path {project_dir}/output -o {output_path}`.
  - Reference doc: look for `v1_docs_path/references/huisstijl.docx`. If exists, add `--reference-doc {path}`. If not, proceed without (warn in log).
  - Output filename scheme: scan existing files in `output/` matching `FDS-v*.docx`, determine next minor version. Pattern: `FDS-v{major}.{minor}_{mode}_{language}.docx`. Start at v1.0. Each subsequent run with same mode+language increments minor. Example: FDS-v1.0_draft_nl.docx, FDS-v1.1_draft_nl.docx.
  - Return the filename (not full path).

- `def list_export_versions(project_dir: Path) -> list[dict]`:
  - Scan `project_dir/output/` for `*.docx` files.
  - Parse filename pattern: `{TYPE}-v{version}_{mode}_{lang}.docx`.
  - Return list of dicts with: filename, doc_type, mode, language, version, created_at (file mtime), size_bytes (file size).
  - Sort by created_at descending (most recent first).

- `async def render_diagrams(project_dir: Path, on_step: Callable) -> None`:
  - on_step("Diagrammen renderen", 3): Check if `mmdc` (mermaid-cli) is available via `shutil.which("mmdc")`.
  - If available: scan assembled markdown for ```mermaid blocks, write each to temp .mmd file, run `mmdc -i {file}.mmd -o {file}.png`, replace markdown blocks with `![diagram](diagram-N.png)`.
  - If not available: log warning, skip diagram rendering (graceful degradation per export.md). No error — just skip.

**Step 1: Create backend/app/api/export.py**

Define router: `APIRouter(prefix="/api/projects/{project_id}/export", tags=["export"])`

Endpoints:

1. `GET /readiness?mode=draft` — Check assembly readiness
   - Load project from DB, get project_dir from `_get_project_dir(project_id)`.
   - Call `check_assembly_readiness(project_dir, mode)`.
   - Return `AssemblyReadinessSchema`.

2. `GET /pandoc-status` — Check Pandoc installation
   - Call `detect_pandoc()`.
   - Return `PandocStatusSchema(installed=found, version=version)`.

3. `GET /stream` — SSE streaming endpoint for assembly + export pipeline
   - Query params: `mode: str = "draft"`, `language: str = "nl"`.
   - Import `EventSourceResponse` from `sse_starlette.sse`.
   - Pre-flight: check Pandoc installed. If not, return HTTP 422 with `{"detail": "Pandoc niet gevonden", "fix_hint": "brew install pandoc"}`.
   - Pre-flight: check readiness. If final mode and unreviewed phases, return HTTP 422 with unreviewed list.
   - Define `async def event_generator()`:
     - 4 steps total: steps = ["Cross-referenties oplossen", "Secties samenvoegen", "DOCX genereren", "Diagrammen renderen"]
     - For each step: yield `{"event": "step_start", "data": json.dumps({"step": i, "name": step, "total_steps": 4})}`.
     - Step 0+1: call `assemble_fds(project_dir, language, mode, v1_docs_path, on_step_callback)`.
     - Step 2: call `export_to_docx(assembled_path, project_dir, mode, language, v1_docs_path, on_step_callback)`.
     - Step 3: call `render_diagrams(project_dir, on_step_callback)` — gracefully skips if mmdc not available.
     - After each step completes: yield `{"event": "step_done", "data": json.dumps({"step": i})}`.
     - On complete: yield `{"event": "complete", "data": json.dumps({"artifact_filename": filename})}`.
     - Between steps: check `await request.is_disconnected()`. If True, break and yield `{"event": "cancelled", "data": "{}"}`.
     - On exception: yield `{"event": "error", "data": json.dumps({"step": i, "name": step, "message": str(e)})}`.
   - Return `EventSourceResponse(event_generator())`.

4. `GET /versions` — List versioned artifacts
   - Call `list_export_versions(project_dir)`.
   - Return `ExportVersionListResponse`.

5. `GET /download/{filename}` — Download a specific artifact
   - Validate filename (no path traversal: reject if `..` in filename or `/` in filename).
   - Construct path: `project_dir / "output" / filename`.
   - If not exists: 404.
   - Return `FileResponse(path, filename=filename, media_type="application/vnd.openxmlformats-officedocument.wordprocessingml.document")`.

Helper: `def _get_project_dir(project_id: int) -> Path` — same pattern as documents.py.
