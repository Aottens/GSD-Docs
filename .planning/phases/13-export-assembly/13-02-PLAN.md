**Step 3: Create backend/app/services/sds_service.py**
Implement SDS scaffolding service following generate-sds.md workflow:

- `def load_catalog(project_dir: Path) -> list[dict] | None`:
  - Look for CATALOG.json at `project_dir/references/typicals/CATALOG.json`.
  - If not found, return None (skeleton mode).
  - Parse JSON, return list of typical objects.

- `def extract_equipment_modules(project_dir: Path, v1_docs_path: Path) -> list[dict]`:
  - Load fds-structure.json from v1_docs_path.
  - Scan `project_dir/.planning/phases/` for equipment module phase directories (pattern: `NN-em-*` or `NN-equipment-*`).
  - For each module dir, extract from SUMMARY.md files: module name, I/O counts (count DI/DO/AI/AO mentions), keywords (extract nouns from module name and content headings), state count (count stateDiagram states).
  - Return list of dicts: `{"name": str, "io": {"DI": int, "DO": int, "AI": int, "AO": int}, "keywords": set[str], "state_count": int}`.
  - If no equipment modules found, return empty list.

- `def score_typical(module: dict, typical: dict) -> float`:
  - **I/O match (40%)**: For each I/O type (DI, DO, AI, AO), compare module count vs typical interface count. `io_score = 40 * (1 - min(total_diff / max(total_module_io, 1), 1))` where total_diff = sum of abs differences per type.
  - **use_cases Jaccard similarity (30%)**: `module_kw = set(module["keywords"])`. `typical_uc = set(word.lower() for uc in typical.get("use_cases", []) for word in uc.split())`. Jaccard = `|intersection| / |union|` if union is non-empty, else 0. `keyword_score = jaccard * 30`.
  - **State count (20%)**: `state_diff = abs(module["state_count"] - len(typical.get("states", [])))`. Score: 0 diff = 20, 1 = 15, 2 = 10, else 5.
  - **Category (10%)**: `category_score = 10 if typical.get("category", "").lower() in {kw.lower() for kw in module["keywords"]} else 0`.
  - Return sum of all four scores (0-100 range).

- `def classify_confidence(score: float) -> tuple[str, str]`:
  - Return `(confidence_level, status)`:
    - score >= 70: ("HIGH", "matched")
    - 40 <= score < 70: ("MEDIUM", "low_confidence")
    - 0 < score < 40: ("LOW", "new_typical_needed")
    - score == 0: ("UNMATCHED", "new_typical_needed")

- `async def scaffold_sds(project_dir: Path, v1_docs_path: Path) -> dict`:
  - Extract equipment modules.
  - Load catalog (returns None if missing).
  - If catalog is None (skeleton mode): for each module, create TypicalMatchSchema with `confidence=0, confidence_level="UNMATCHED", status="new_typical_needed", matched_typical=None`.
  - If catalog exists: for each module, score against all typicals, pick best match. Build TypicalMatchSchema with full match_detail including breakdown scores and reason text.
  - For unmatched/low modules: populate `closest_match` with the highest-scoring typical name even if below threshold.
  - Populate `cli_hint` as `/doc:generate-sds --refine {module_name}`.
  - Build reason strings: For matched: "I/O match {io_score}/40, keywords {keyword_score}/30, states {state_score}/20, category {category_score}/10". For unmatched: describe which dimension scored lowest.
  - Persist results to `project_dir/output/sds-results.json` (JSON serialized SdsResultsResponse).
  - Return the results dict.
