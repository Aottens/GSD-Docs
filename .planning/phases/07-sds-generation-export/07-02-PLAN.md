  <name>Task 2: Create DOCX export workflow with Mermaid rendering and Pandoc conversion</name>
  <files>

  </files>
  <action>
Create comprehensive export workflow following established workflow patterns (see complete-fds.md workflow for reference — detailed step-by-step with numbered steps).

**Export Pipeline Steps (10 steps):**

**Step 1: Validate Prerequisites**
- Check Pandoc installed: `pandoc --version`. Extract version number. Compare against recommended version (3.9+). If different, warn but continue.
- Check mermaid-cli installed: `mmdc --version`. If not found and --skip-diagrams NOT set, warn and auto-enable --skip-diagrams with message.
- Check huisstijl.docx exists at `gsd-docs-industrial/references/huisstijl.docx`. If missing, warn: "Corporate style template not found. Export will use Pandoc defaults. Place your template at gsd-docs-industrial/references/huisstijl.docx for branded output."
- Check input document exists. If not specified, scan `output/` for latest assembled FDS or SDS document (by modification time).
- Create `export/` directory if it doesn't exist.
- Create `diagrams/generated/` directory if it doesn't exist.

**Step 2: Parse Document Metadata**
- Read YAML frontmatter from input markdown (title, project, version, date, type FDS/SDS, based_on for SDS).
- Determine output filename: `{type}-{project}-v{version}.docx` (or with `-DRAFT` suffix if --draft).
- Log: `DOC > Export: {input} → {output}`

**Step 3: Extract and Render Mermaid Diagrams** (skip if --skip-diagrams)
- Scan input markdown for all ```mermaid code blocks.
- For each diagram:
  a. Analyze complexity: count nodes (lines matching `/^\s*[A-Za-z0-9_]+[\[\(\{]/`), count edges (lines with `-->`, `---`, `==>`, `===`).
  b. If nodes > 100 (hard limit): route to ENGINEER-TODO.md with section reference, skip render. Replace block with `[DIAGRAM DEFERRED: Exceeds 100-node complexity limit. See ENGINEER-TODO.md]` (EXPT-06).
  c. If nodes > 40 (soft limit): log warning but proceed with render.
  d. Render with mermaid-cli: `mmdc -i <temp-file> -o diagrams/generated/diagram-{index}.png -t neutral -b white --scale 2 --width 1200`
  e. Set 60-second timeout per diagram. On timeout: route to ENGINEER-TODO.md, replace with placeholder.
  f. On render success: replace ```mermaid block in markdown with `![{caption}](diagrams/generated/diagram-{index}.png)` image reference.
  g. On render failure: log error, route to ENGINEER-TODO.md, replace with `[DIAGRAM FAILED: {error}. See ENGINEER-TODO.md]`.
- Log: `DOC > Diagrams: {success} rendered, {deferred} deferred, {failed} failed`

**Step 4: Handle External Diagrams**
- Scan for image references pointing to `diagrams/external/` directory.
- If PNG exists at referenced path: keep reference (Pandoc will embed).
- If PNG does NOT exist: replace with text reference `[EXTERNAL DIAGRAM: {path}]` per user decision (both embed and reference supported).
- Log count of external diagrams found/embedded.

**Step 5: Prepare Pandoc Input**
- Write processed markdown (with Mermaid blocks replaced by image references) to temporary file.
- Ensure all image paths are relative to working directory (Pandoc resolves from CWD).

**Step 6: Build Pandoc Command**
Build the Pandoc conversion command:
```
pandoc {input}
  --from markdown+yaml_metadata_block+pipe_tables+grid_tables+implicit_figures
  --to docx
  --standalone
  --toc --toc-depth=3
  --number-sections
  --resource-path=.
  -o {output}
```

Add conditional flags:
- If huisstijl.docx exists: add `--reference-doc=gsd-docs-industrial/references/huisstijl.docx` (EXPT-02, EXPT-03)
- If NOT --draft: add `--metadata lof:true` and `--metadata lot:true` for list of figures and list of tables (or use Pandoc's `--lof --lot` if version supports it)
- If --draft: add `--metadata draft:true` in YAML header

**Step 7: Execute Pandoc Conversion**
- Run Pandoc command. Capture stdout and stderr.
- On success: verify output file exists and size > 10KB (sanity check — empty DOCX is ~8KB with styles).
- On failure: display error, suggest common fixes (version mismatch, reference doc corruption, invalid markdown).
- Log: `DOC > Pandoc: Conversion complete ({file-size})`

**Step 8: Generate ENGINEER-TODO.md for Deferred Diagrams**
- If any diagrams were deferred or failed in Step 3:
  - Read existing ENGINEER-TODO.md if present (may have entries from /doc:complete-fds).
  - Append new entries under "## Export: Deferred Diagrams" section.
  - Each entry: Section reference, Diagram type (state/flow/sequence), Issue description, Priority (HIGH for deferred, MEDIUM for failed), Suggestion.
- If no deferred diagrams: skip.

**Step 9: Update STATE.md**
- Record export completion in STATE.md: Last export date, document exported, format (DOCX), draft flag, diagrams rendered/deferred.

**Step 10: Display Summary**
```
DOC > ════════════════════════════════════════
DOC > Export Complete
DOC > ════════════════════════════════════════
DOC > Document: {input-file}
DOC > Output:   {output-file} ({size})
DOC > Format:   DOCX with {huisstijl/default} styling
DOC > Diagrams: {N} rendered, {M} deferred, {K} external
DOC > Draft:    {yes/no}
DOC > ════════════════════════════════════════
```

If --draft: add note "Draft export — list of figures/tables omitted. Run without --draft for final export."
If deferred diagrams: add note "See ENGINEER-TODO.md for {N} diagrams requiring manual creation."
If no huisstijl.docx: add note "Export used Pandoc defaults. Place huisstijl.docx in references/ for corporate styling."

**Error handling throughout:**
- Every external tool call (pandoc, mmdc) wrapped in error check.
- Fail fast with clear error messages, no retry logic per research recommendation.
- Clean up temporary files on exit (processed markdown, temp mermaid files).

**Important implementation notes:**
- The workflow file is MARKDOWN with step-by-step instructions for Claude to follow, NOT a shell script. Follow the same pattern as complete-fds.md workflow.
- Use the same lean-command + detailed-workflow separation pattern established in Phase 1.
- Target ~800-1200 lines for the workflow (comparable to complete-fds.md at 1536 lines but simpler pipeline).
