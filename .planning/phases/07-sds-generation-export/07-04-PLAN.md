Include 8-10 realistic control module typicals covering common industrial automation building blocks:

1. **FB_AnalogIn** — Category: Input Processing. 4-20mA analog input with scaling, high/low alarms, signal quality monitoring. Inputs: RawValue (INT), ScaleMin (REAL), ScaleMax (REAL), HighLimit (REAL), LowLimit (REAL). Outputs: ScaledValue (REAL), HighAlarm (BOOL), LowAlarm (BOOL), SignalFault (BOOL). Use cases: Temperature sensors, Pressure transmitters, Flow meters, Level sensors.

2. **FB_DigitalIn** — Category: Input Processing. Digital input with debouncing, inversion option, pulse counting. Inputs: RawInput (BOOL), DebounceTime (TIME), Invert (BOOL). Outputs: ProcessedValue (BOOL), PulseCount (DINT), FaultState (BOOL). Use cases: Limit switches, Proximity sensors, Push buttons.

3. **FB_MotorDOL** — Category: Motor Control. Direct-on-line motor starter with thermal protection, start/stop control, running feedback monitoring. Inputs: CmdStart (BOOL), CmdStop (BOOL), FeedbackRunning (BOOL), ThermalOK (BOOL). Outputs: ContactorOut (BOOL), MotorRunning (BOOL), MotorFault (BOOL), RunHours (REAL). States: IDLE, STARTING, RUNNING, STOPPING, FAULT. Use cases: Conveyor motors, Pump motors (constant speed), Fan motors.

4. **FB_MotorVFD** — Category: Motor Control. Variable frequency drive motor control with speed setpoint, ramp times, frequency feedback. Inputs: CmdStart (BOOL), CmdStop (BOOL), SpeedSetpoint (REAL), RampUpTime (TIME), RampDownTime (TIME), FrequencyFeedback (REAL), DriveReady (BOOL), DriveFault (BOOL). Outputs: DriveEnable (BOOL), SpeedReference (REAL), MotorRunning (BOOL), AtSpeed (BOOL), DriveFaultOut (BOOL). States: IDLE, ACCELERATING, RUNNING, DECELERATING, FAULT. Use cases: Mixer agitators, Variable speed pumps, Dosing systems.

5. **FB_ValveOnOff** — Category: Valve Control. On/off valve with feedback monitoring, stroke time supervision, interlock support. Inputs: CmdOpen (BOOL), CmdClose (BOOL), FbkOpen (BOOL), FbkClosed (BOOL), Interlock (BOOL), StrokeTime (TIME). Outputs: ValveOpen (BOOL), ValveClosed (BOOL), ValveTransit (BOOL), ValveFault (BOOL). States: CLOSED, OPENING, OPEN, CLOSING, FAULT. Use cases: Isolation valves, Drain valves, Product routing.

6. **FB_ValveModulating** — Category: Valve Control. Modulating control valve with position setpoint, feedback, PID option. Inputs: PositionSetpoint (REAL), PositionFeedback (REAL), CmdOpen (BOOL), CmdClose (BOOL), PIDEnable (BOOL). Outputs: ValveOutput (REAL), PositionError (REAL), ValveFault (BOOL). Use cases: Flow control valves, Temperature control valves, Pressure control valves.

7. **FB_PIDControl** — Category: Process Control. Standard PID controller with auto/manual mode, anti-windup, output limiting. Inputs: Setpoint (REAL), ProcessValue (REAL), Kp (REAL), Ti (TIME), Td (TIME), OutputMin (REAL), OutputMax (REAL), ManualMode (BOOL), ManualOutput (REAL). Outputs: ControlOutput (REAL), Error (REAL), InAuto (BOOL). Use cases: Temperature control, Level control, Pressure control, Flow control.

8. **FB_WeighScale** — Category: Measurement. Weighing scale with tare, calibration, dosing integration. Inputs: RawWeight (REAL), TareCmd (BOOL), CalibrationFactor (REAL). Outputs: NetWeight (REAL), GrossWeight (REAL), Stable (BOOL), Overload (BOOL). Use cases: Batch dosing, Material handling, Packaging.

9. **FB_AlarmHandler** — Category: System. Alarm management with priority levels, acknowledgment, logging interface. Inputs: AlarmActive (BOOL), AlarmPriority (INT), AcknowledgeCmd (BOOL). Outputs: AlarmState (INT), Acknowledged (BOOL), AlarmTimestamp (DT). Use cases: Equipment alarms, Process alarms, Safety alarms.

10. **FB_SequenceCtrl** — Category: Sequence Control. Step sequence controller with step management, transition logic, hold/resume. Inputs: CmdStart (BOOL), CmdHold (BOOL), CmdResume (BOOL), CmdAbort (BOOL), TransitionOK (BOOL). Outputs: CurrentStep (INT), StepActive (BOOL), SequenceComplete (BOOL), SequenceHeld (BOOL). States: IDLE, RUNNING, HELD, COMPLETE, ABORTED. Use cases: Batch sequences, CIP cleaning, Startup sequences.

Each typical must have: id, type ("Control Module"), category, description, interfaces (inputs/outputs with name+dataType), use_cases array, states array (where applicable), documentation (set to null — no docs in pilot).
