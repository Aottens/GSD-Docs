# Phase 10.1 Plan 02: Frontend Discussion Behavior Rework Summary

**One-liner:** Implemented new SSE event handling (decision_captured, topic_boundary, completion_card, check_in), chip-as-starting-point question cards, and confirm/reject decision workflow with real-time capture in summary panel.

## What Was Built

Updated frontend components to match the reworked discussion behavior design from Phase 10.1:

1. **TypeScript types and SSE event handling**: Expanded event types to cover decision_captured, topic_boundary, completion_card, check_in events. Added Decision interface fields for question, confirmed status, notes, and timestamp. Created CompletionData and TopicBoundaryData interfaces.

2. **useDiscussionStream hook enhancements**: Added decisions, completionData, and currentTopic state management. Implemented SSE event handlers for all new event types with proper state updates and chat message creation. Added confirmDecision, rejectDecision, and addDecisionNote callbacks.

3. **QuestionCard chip-follow-up behavior**: Changed chip clicks to send messages but keep card open (awaiting follow-up from AI). Added isActive prop to control card state. Show checkmark on selected chip without closing the card. Freeform text input always visible.

4. **SummaryPanel confirm/reject UI**: Replaced inline editing with confirm/reject/notes pattern. Each unconfirmed decision shows Confirm and Reject buttons. Confirmed decisions display green checkmark indicator. Add notes link expands text input for context.

5. **ChatPanel integration**: Wired decisions state and callbacks from useDiscussionStream to SummaryPanel. Show currentTopic badge in header. Display completionData message when available. Pass sendMessage directly to MessageList as onAnswer prop. SummaryPanel visible whenever there are decisions.

## Key Technical Decisions

1. **Chip-as-starting-point pattern**: Per user decision, chip selections now trigger follow-up questions by sending the chip text as a message but keeping the card open. This enables deeper exploration of each topic through conversational follow-ups.

2. **Reject reopens discussion**: Rejecting a decision removes it from the panel and sends "[Beslissing afgewezen] {original_question}" to the chat, restarting the discussion thread rather than allowing inline editing.

3. **Freeform input visibility**: Kept freeform text input always visible below chips for simpler interface. Engineers can always type detailed answers instead of clicking chips.

4. **Decision state management**: Decisions tracked in useDiscussionStream hook with confirmed/notes fields. All decision operations (confirm/reject/addNote) managed through callbacks returned from the hook.

## Integration Points

- **useDiscussionStream → ChatPanel**: Hook provides decisions state and callbacks, wired to SummaryPanel props
- **ChatPanel → MessageList → MessageBubble → QuestionCard**: onAnswer prop chain passes sendMessage function to enable chip clicks to send messages through SSE stream
- **SummaryPanel → useDiscussionStream**: Confirm/reject/addNote callbacks trigger decision state updates and message sending
- **SSE events → UI updates**: New event types (decision_captured, topic_boundary, completion_card, check_in) create appropriate messages in chat and update relevant state
