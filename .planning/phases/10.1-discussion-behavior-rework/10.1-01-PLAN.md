    Create `backend/app/services/conversation_state.py` with:

    1. `ConversationPhase` enum: `topic_selection`, `discussion`, `check_in`, `completion`
    2. `ConversationState` dataclass with fields:
       - `phase: ConversationPhase` (current conversation phase)
       - `selected_topics: list[str]` (topics engineer chose to discuss)
       - `completed_topics: list[str]` (topics fully discussed)
       - `current_topic: Optional[str]` (active topic being discussed)
       - `questions_in_current_topic: int` (counter for 4-question check-in rhythm)

       - `deferred_ideas: list[dict]` (scope creep captures)
       - `is_foundation: bool` (True if Foundation/intake phase)
    3. State transition methods:
       - `start_topic_selection()` -> sets phase to topic_selection
       - `confirm_topics(selected: list[str], discretion: list[str])` -> transitions to discussion, sets selected_topics
       - `start_topic(topic: str)` -> sets current_topic, resets question counter
       - `increment_question()` -> bumps counter, returns True if >= 4 (check-in time)
       - `complete_topic()` -> moves current_topic to completed_topics, sets current_topic=None
       - `start_completion()` -> transitions to completion phase
       - `next_topic() -> Optional[str]` -> returns next unprocessed topic from selected_topics
       - `all_topics_complete() -> bool` -> True if all selected_topics are in completed_topics
    4. `from_summary_data(data: dict) -> ConversationState` classmethod to reconstruct from DB JSON
    5. `to_summary_data() -> dict` method to serialize for DB storage
    6. `detect_foundation_phase(phase_number: int, phase_goal: str) -> bool` standalone function:
       - Returns True if phase_number == 1 OR phase_goal contains "foundation", "intake", "scope" keywords
       - Per user decision and v1.0 discuss-phase.md Step 2.2

  <action>
    Create `backend/app/services/structured_output_parser.py` with:

    1. `parse_question_card(text: str) -> Optional[dict]`:
       - Uses regex to find `<question>...</question>` and optional `<options><option>...</option></options>` tags
       - Returns `{"question": str, "options": list[str]}` or None if no question tag found
       - Must handle multi-line content within tags
       - Must handle missing options tag (freeform-only question)
       - Per research: use regex for simple tags, not full XML parser (lxml overkill for this)

    2. `parse_topic_boundary(text: str) -> Optional[dict]`:
       - Finds `<topic_boundary topic="..." status="..." />`
       - Returns `{"topic": str, "status": str}` or None

    3. `parse_completion_signal(text: str) -> Optional[str]`:
       - Finds `<completion_signal>...</completion_signal>`
       - Returns the signal text or None

    4. `strip_xml_tags(text: str) -> str`:
       - Removes all recognized XML tags from text, returning plain conversational content
       - Used for the message_delta events (frontend sees clean text, structured data sent separately)

    5. `StreamingXMLBuffer` class for incremental parsing during SSE streaming:
       - `feed(chunk: str)` method that accumulates text
       - `extract_events() -> list[dict]` that returns any complete structured events found in buffer
       - Handles partial tags across chunk boundaries (e.g., `<quest` in one chunk, `ion>text</question>` in next)
       - Returns events as dicts: `{"type": "question_card", "data": {...}}`, `{"type": "topic_boundary", "data": {...}}`, `{"type": "completion_signal", "data": {...}}`
       - After extracting, removes consumed content from buffer
       - `get_text_content() -> str` returns any non-tag text accumulated (for message_delta events)

    Update `backend/app/schemas/conversation.py`:
    6. Add Pydantic models for the new SSE event types:
       - `QuestionCardEvent(question: str, options: list[str] = [])`
       - `DecisionCapturedEvent(topic: str, question: str, decision: str, confirmed: bool = False, timestamp: str)`
       - `TopicBoundaryEvent(topic: str, status: str)` (status: "complete" or "starting")
       - `CompletionSignalEvent(message: str)`
       - `CheckInEvent(topic: str, questions_asked: int)`
  </action>
