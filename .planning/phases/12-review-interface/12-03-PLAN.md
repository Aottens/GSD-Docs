```typescript
  switch (reviewStatus) {

    case 'opmerking': return 'Opmerking toegevoegd'
    case 'afgewezen': return 'Afgewezen'
```

4. Inside the `OutlineNode` component function body, add at the top:
```typescript
// Try to get review context — may not exist if not wrapped in ReviewProvider
let reviewStatus: SectionReview['status'] | undefined
try {
  const { reviews } = useReviewContext()
  reviewStatus = reviews[node.id]?.status
} catch {
  // Not inside ReviewProvider — no review status available
  reviewStatus = undefined
}
```

NOTE: Using try/catch around useReviewContext is needed because OutlineNode may render outside ReviewProvider context (e.g., when DocumentsTab hasn't loaded verification data yet, or in other contexts). The alternative is to make the hook return null when outside provider, but since Plan 01 defined it to throw, we catch here.

5. Replace the status icon rendering section. Where it currently shows `{getStatusIcon(node.status)}`, replace with:
```tsx
{/* Status icon — review status takes precedence */}
{reviewStatus ? (
  <TooltipProvider>

      <TooltipTrigger asChild>
        <span>{getReviewIcon(reviewStatus)}</span>
      </TooltipTrigger>
      <TooltipContent>{getReviewTooltip(reviewStatus)}</TooltipContent>
    </Tooltip>
  </TooltipProvider>
) : (
  getStatusIcon(node.status)
```

This makes review status from ReviewContext take precedence over backend node.status per UI-SPEC Interaction Contract 5.
