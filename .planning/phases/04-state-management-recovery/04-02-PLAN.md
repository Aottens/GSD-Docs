Create gsd-docs-industrial/workflows/status.md (~300-400 lines) wrapped in `<workflow>` tags. This is the core rendering logic for /doc:status.

**Step 1: Load Project State**

Read three data sources:
1. `.planning/STATE.md` — Extract Current Operation section (command, phase, wave, status), Decisions, Blockers, Session Continuity
2. `.planning/ROADMAP.md` — Parse all phase entries (number, name, requirements count, status)
3. `.planning/PROJECT.md` — Extract project name, type, language

If `.planning/` directory does not exist: display error box "No project found. Run /doc:new-fds to create a project." and stop.

**Step 2: Calculate Progress from Filesystem**

For EACH phase listed in ROADMAP.md:
1. Find phase directory: `.planning/phases/{NN}-{name}/`
2. Count PLAN.md files: `{NN}-*-PLAN.md` (total plans)
3. Count SUMMARY.md files: `{NN}-*-SUMMARY.md` (completed plans — SUMMARY.md is the completion proof)
4. Check for VERIFICATION.md: `{NN}-VERIFICATION.md` or `VERIFICATION.md` (verification done)
5. Check for partial writes using the 4-heuristic cascade from Plan 01:
   - For each CONTENT.md without matching SUMMARY.md: partial (missing_summary)
   - For each CONTENT.md < 200 chars: partial (content_too_short)
   - For each CONTENT.md containing [TO BE COMPLETED]: partial (completion_marker)
   - Abrupt ending check (MEDIUM confidence)

Derive phase status:
- `Verified` — VERIFICATION.md exists with `Status: PASS`
- `Gaps Found` — VERIFICATION.md exists with `Status: GAPS_FOUND`
- `Written` — All plans have SUMMARY.md but no VERIFICATION.md yet
- `In Progress` — Some plans have SUMMARY.md, or Current Operation targets this phase
- `Planned` — PLAN.md files exist but no SUMMARY.md files
- `Pending` — No PLAN.md files in phase directory (or directory doesn't exist)
- `Blocked` — STATE.md shows phase as BLOCKED

Calculate overall completion: `sum(completed_plans) / sum(total_plans) * 100` (only count phases that HAVE plans; pending phases with 0 plans are excluded from denominator).

**Step 3: Display Progress**

Display banner:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 DOC > PROJECT STATUS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

Display project info:
```
Project: {project_name}
Type: {type} | Language: {language}
```

Display overall progress bar:
```
Progress: ████████████████░░░░░░░░░░░░░░ 53%
```

Use `█` for filled and `░` for empty. 30-character bar width. Calculate filled = round(percent / 100 * 30).

**Step 4: Display Phase Table**

Display per-phase table:
```
| Phase | Name                              | Plans  | Status       |
|-------|-----------------------------------|--------|--------------|
| 1     | Framework Foundation              | 4/4    | ✓ Verified   |
| 2     | Discuss + Plan Commands           | 4/4    | ✓ Verified   |
| 3     | Write + Verify (Core Value)       | 3/5    | ⚙ In Progress|
| 4     | State Management + Recovery       | -/-    | ○ Pending    |
| 5     | Complete-FDS + Standards          | -/-    | ○ Pending    |
| 6     | Knowledge Transfer + Review       | -/-    | ○ Pending    |
| 7     | SDS Generation + Export           | -/-    | ○ Pending    |
```

Status indicators: `✓` (verified), `⚙` (in progress/written/gaps), `○` (pending/planned), `✗` (blocked).

For phases with 0 plans, show `-/-` for plan count.

**Step 5: Display Active Phase Detail**

Find the active phase (first phase that is not Verified and not Pending — i.e., In Progress, Written, Gaps Found, Planned, or Blocked).

If active phase found, display:
```
Active Phase: {N} — {Phase Name}

| Plan    | CONTENT.md | SUMMARY.md | Status    |
|---------|------------|------------|-----------|
| {NN}-01 | ✓ 2.3KB   | ✓ 142w    | Complete  |
| {NN}-02 | ✓ 1.8KB   | ✗ missing  | Partial   |
| {NN}-03 | ✗ missing  | ✗ missing  | Pending   |

VERIFICATION.md: {Yes (PASS) | Yes (GAPS_FOUND, cycle 1/2) | No}
```

**Step 6: Display Partial Write Warnings**

If any partial writes detected in the active phase:
```
⚠ Partial Writes Detected:

- {plan-id}: {reason} — Run /doc:resume or /doc:write-phase {N}
- {plan-id}: {reason} — Run /doc:resume or /doc:write-phase {N}
```

**Step 7: Display Current Operation (if active)**

If STATE.md Current Operation has status IN_PROGRESS:
```
Current Operation: {command} phase {phase}
  Wave: {wave}/{wave_total}
  Plans done: {plans_done}
  Plans pending: {plans_pending}
  Started: {started}
```

**Step 8: Derive and Display Next Action**

Logic to determine recommended next action (checked in this order):
1. If Current Operation status is IN_PROGRESS → "Resume interrupted operation" → `/doc:resume`
2. If any phase has partial writes → "Complete partial writes" → `/doc:write-phase {N}` or `/doc:resume`
3. If any phase has status Blocked → "Resolve blocked phase" → "See .planning/ENGINEER-TODO.md"
4. If any phase has status Gaps Found → "Close gaps in phase {N}" → `/doc:plan-phase {N} --gaps`
5. If any phase has status Written (not verified) → "Verify phase {N}" → `/doc:verify-phase {N}`
6. If any phase has status Planned (has plans, nothing written) → "Write phase {N}" → `/doc:write-phase {N}`
7. If any phase has CONTEXT.md but no plans → "Plan phase {N}" → `/doc:plan-phase {N}`
8. If next unstarted phase exists → "Discuss phase {N}" → `/doc:discuss-phase {N}`
9. If all phases verified → "Assemble FDS" → `/doc:complete-fds`

Display:
```
───────────────────────────────────────────────────────────────

Next: {action description} ({context: e.g., "3 plans ready, 0 written"})

  {command}

───────────────────────────────────────────────────────────────
```

- Use SUMMARY.md existence as the ONLY completion proof for plan completion
