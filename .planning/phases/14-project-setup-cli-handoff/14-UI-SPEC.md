## Typography

Matches established patterns from ProjectOverview.tsx, FaseringTab.tsx, and Step4ReferenceUpload.tsx.

4-size scale, 2-weight scale. CLI mono text is a `font-mono` modifier on the Body size — not a separate size entry.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 30px (text-3xl) | 600 (semibold) | 1.2 | Project name heading in overview card |
| Heading | 24px (text-2xl) | 600 (semibold) | 1.2 | Wizard step heading, tab section headings |
| Subheading | 18px (text-lg) | 600 (semibold) | 1.3 | Card section titles (e.g. "Setup status", "Snelkoppelingen") |
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Checklist row labels, metadata text, description copy, file counts, muted helper text (`text-muted-foreground`), CLI command text (`font-mono` modifier) |

**Modifiers on Body (14px / 400):**
- Captions and muted helper text: add `text-muted-foreground` class — no separate size
- CLI command text in `CliCommandBlock`: add `font-mono` class — no separate size

## Component Inventory

New components introduced in this phase, with reuse guidance.

### New: `DocTypeChecklist` (Wizard Step 4 replacement)

Replaces `Step4ReferenceUpload`. Renders a vertical list of doc-type rows specific to project type (A/B/C/D).

**Row anatomy (left to right):**
1. Status icon (16px lucide icon: `CheckCircle2` green / `Clock` amber / `Circle` muted)
2. Doc type label (text-sm, font-medium) — Dutch label from backend config
3. File count badge (text-xs, muted) — e.g. "2 bestanden" or "—"
4. Inline upload button (Button variant="outline" size="sm") — triggers FileUploadZone popover or inline input
5. "Niet beschikbaar" toggle (Checkbox or Switch) — grays out entire row when checked

**Row states:**
- Default: white bg, all controls enabled
- Uploaded: green status icon, file count shown, upload button changes to "Vervangen / Toevoegen"
- Marked unavailable: entire row at `opacity-50`, label strikethrough, toggle checked
- Hover: `bg-muted/50` row highlight

**Container:** `space-y-2` between rows, wrapped in a `Card` with `p-6`.

### New: `SetupStatusSection` (inside ProjectOverview)

A new section added to the existing overview card after the Metadata grid, separated by `<Separator />`.

**Section heading:** "Setup status" at text-lg / semibold.

**Content:**
- Doc-type coverage checklist (same row pattern as `DocTypeChecklist` but read-only: no upload button, no toggle)
- `CliCommandBlock` below the checklist showing the context-aware CLI command

**CLI command logic (display only):**
- No phases scaffolded yet: `/doc:new-fds`
- Phases present but not all complete: phase-specific `/doc:*` command per `STATUS_CLI_COMMANDS`

### New: `DocCoverageSection` (in Referenties tab)

A collapsible section placed above the existing file browser in the Referenties tab.

**Heading:** "Document dekking" — collapsed by default (Collapsible component, existing in ui/).

**Content when expanded:** Same read-only checklist rows as `SetupStatusSection`.

**Doc-type upload flow:** When engineer uploads a file via the existing FileUploadZone in the Referenties tab, a `Select` dropdown (shadcn select) appears immediately after file selection, prompting for doc-type classification. Label: "Documenttype" with placeholder "Selecteer type...". Submit button: "Uploaden".

### Reused: `CliCommandBlock`

Defined locally in FaseringTab.tsx. Extract to `frontend/src/features/timeline/components/CliCommandBlock.tsx` for reuse in `SetupStatusSection`. No visual changes.

### Reused: `FileUploadZone`

Used as-is for inline upload within `DocTypeChecklist` rows. No visual changes.

## Interaction Contracts

### IC-1: Wizard Step 4 — Doc-type checklist

- Checklist renders immediately on step mount; rows derived from project type (A/B/C/D)
- Inline upload button opens native file picker (or drags into row); does NOT navigate away from wizard
- After file selection, upload starts immediately (XHR progress via `useFileUpload`); row status icon changes to spinner during upload, then green `CheckCircle2` on success
- "Niet beschikbaar" toggle: checking grays row + records decision; unchecking restores row to default (Ontbreekt)
- Step can be completed (wizard "Volgende" enabled) regardless of coverage — no doc type is required
- Step "Overslaan" action: skip link below checklist, labeled "Sla referenties over" — navigates to wizard complete step

### IC-2: Setup status — ProjectOverview

- Section loads with React Query (same polling interval as Phase 11: 5000ms) — skeleton shown while loading
- Checklist rows: read-only, no upload or toggle controls
- CLI command: always visible in `CliCommandBlock`; copy button copies to clipboard; toast confirms "Gekopieerd!"
- After CLI setup writes files to disk, React Query polling detects change and re-renders section automatically — no manual refresh needed

### IC-3: Late-arriving documents — Referenties tab

- "Document dekking" section: collapsed by default, expand caret top-right of section header
- When engineer uploads a file: `Select` dropdown appears inline below the file name before the upload begins
- If engineer skips the dropdown (selects no type): file uploads as "Zonder type" — no doc-type assignment
- After upload with type assigned: doc-type coverage section updates automatically via React Query invalidation
- Note text shown below coverage section: "Nieuw document toegevoegd — voer /doc:discuss-phase N uit om opnieuw te analyseren." — static text, no action button, text-sm text-muted-foreground

### IC-4: Backend setup-state endpoint polling

- `GET /api/projects/{id}/setup-state` polled via React Query at 5000ms interval
- On network error: show existing error pattern — "Kon setup status niet laden" with retry affordance
- Endpoint returns doc-type coverage array; empty array = no coverage info yet (show skeleton checklist with all rows in "Ontbreekt" state)

## States & Feedback

| State | Treatment |
|-------|-----------|
| Loading — setup status | Skeleton rows (3 rows, h-10 each) matching checklist row height |
| Loading — doc-type upload | Progress bar within row, same pattern as FileUploadZone progress |
| Success — file uploaded | Toast: "Bestand geüpload" (sonner, default duration) |
| Success — copy CLI command | Toast: "Gekopieerd!" (sonner, 2000ms) |
| Error — upload failed | Toast: "Upload mislukt — probeer opnieuw" (sonner, destructive variant) |
| Error — setup endpoint | Inline error message with retry link, text-sm text-destructive |
| Empty — no doc types configured | Show generic "Upload referentiedocumenten" placeholder (fallback for unknown project type) |
