1. **api/projects.py** — Add two new endpoints:

   a) `GET /api/doc-types/{project_type}` — Returns `DOC_TYPE_CONFIG[project_type]` as a list of `DocTypeConfigEntry`. No DB query needed. Returns 404 if project_type not in DOC_TYPE_CONFIG. Import `DOC_TYPE_CONFIG` from `config_phases` and `DocTypeConfigEntry` from `schemas.setup_state`.

   b) `GET /api/projects/{project_id}/setup-state` — Returns `SetupStateResponse`. Logic:
      - Load project from DB (404 if not found)
      - Load `DOC_TYPE_CONFIG[project.type.value]` for the project type
      - Query all non-deleted project files with `doc_type IS NOT NULL`, grouped by doc_type: `select(File).where(and_(File.project_id == project_id, File.is_deleted == False, File.doc_type.isnot(None)))`
      - Parse `project.skipped_doc_types` (JSON string or None) into a Python list
      - For each doc type config entry, compute status:
        - If files exist with matching doc_type: status = "present", file_count = count, file_paths = list of absolute storage paths (use `get_absolute_path` from `file_storage` or construct from `settings.UPLOAD_DIR`)
        - If doc type ID is in skipped list: status = "skipped", file_count = 0, file_paths = []
        - Otherwise: status = "missing", file_count = 0, file_paths = []
      - Compute `has_scaffolding`: check if `_get_project_dir(project_id, settings) / ".planning"` directory exists. Reuse `_get_project_dir` logic from `phases.py` (import `get_settings` and construct `Path(settings.PROJECT_ROOT) / str(project_id)`)
      - Compute `next_cli_command`: If not has_scaffolding, return `/doc:new-fds`. If has_scaffolding, find the first phase with `not_started` status using `_derive_phase_status` from phases.py, and call `get_cli_command`. If all phases completed, return None.
      - Return `SetupStateResponse(project_id=project.id, project_name=project.name, project_type=project.type.value, language=project.language.value, project_dir=str(project_dir), doc_types=[...], has_scaffolding=has_scaffolding, next_cli_command=next_cli_command)`
